## 用户故事

**作为** 第三方应用开发者，**我希望** SDK 提供 AK/SK 签名机制，**以便** 我的应用间调用能够通过签名验证请求的完整性和防篡改性。

## 验收标准

1. `AkSignHelper.sign(appId, appSecret)` 生成请求签名
2. 签名算法流程：
    - 生成当前时间戳 timestamp
    - 生成随机数 nonce = MD5(UUID + timestamp)
    - 构建参数 Map: {appId, nonce, timestamp}
    - 按 Key 字典排序拼接为 `appId=xxx&nonce=xxx&timestamp=xxx`
    - 使用 RSA 私钥（appSecret）加密拼接字符串，返回签名
3. 验签方法（预留，当前注释未启用）：
    - RSA 公钥解密签名，还原参数字符串
    - 验证 appId 是否被偷换
    - 验证 timestamp 是否超过 5 分钟过期
    - 使用 Redis 分布式锁验证 nonce 是否重复（防重放攻击）

## 技术实现要点

- 签名使用 RSA 私钥加密，验签使用 RSA 公钥解密
- 字典排序确保参数顺序一致
- nonce 防重放：使用 Redis 分布式锁验证唯一性
- timestamp 有效期 5 分钟，防止签名长期有效
- 验签方法当前被完整注释，依赖 RedisLock 和 AppInfo 缓存，尚未启用
