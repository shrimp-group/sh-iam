## 用户故事

**作为** 第三方应用开发者，**我希望** SDK 提供 JWT 令牌的生成、解析、验证和刷新能力，**以便** 我的应用能够安全地管理用户会话，实现无状态认证。

## 验收标准

1. `JwtUtil.generateToken(UserJwt, secretKey)` 生成 4h 过期的访问令牌
2. `JwtUtil.generateRefreshToken(UserJwt, secretKey)` 生成 7d 过期的刷新令牌
3. `JwtUtil.parseToken(token, secretKey)` 解析令牌返回 UserJwt 对象
4. `JwtUtil.validateToken(token, secretKey)` 验证令牌签名和有效期
5. `JwtUtil.isExpired(token, secretKey)` 检查令牌是否已过期
6. `JwtUtil.refreshToken(token, secretKey)` 刷新令牌
7. `JwtUtil.getTokenRedisKey(token, identifier)` 生成 Redis 会话 Key：`iam:session:{identifier}:{md5(token)}`
8. `JwtUtil.isTokenAboutToExpire(token, secretKey)` 判断令牌是否在 1h 内过期
9. `JwtUtil.isValidTokenWithClockSkew(token, secretKey)` 带时钟偏差容忍（1min）的有效性校验
10. `JwtUtil.safeRefreshToken(token, secretKey)` 安全刷新（仅当令牌有效且未过期时）
11. `JwtUtil.generateSecureSecretKey()` 生成 512 位 Base64 安全随机密钥
12. JWT Claims 包含 userCode、username、nickname、avatar 字段
13. 签名算法使用 HS256
14. 5 种 JWT 异常统一包装为中文描述

## 技术实现要点

- 签名算法：HS256，密钥通过 `Keys.hmacShaKeyFor()` 从 UTF-8 字节生成
- 时钟偏差容忍：`CLOCK_SKEW_TOLERANCE = 60 * 1000L`（1 分钟）
- 生成令牌时 `issuedAt` 提前 1 分钟，容忍客户端时钟偏差
- 访问令牌默认过期：4h，刷新令牌默认过期：7d
- Redis Key 格式：`iam:session:{identifier}:{md5(token)}`
- 异常处理：ExpiredJwtException / MalformedJwtException / UnsupportedJwtException / SignatureException /
  IllegalArgumentException 统一包装
