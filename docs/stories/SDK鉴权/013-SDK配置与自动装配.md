## 用户故事

**作为** 第三方应用开发者，**我希望** 引入 iam-sdk 依赖后，通过简单的配置即可自动装配所有 SDK 组件（过滤器、工具类、配置等），*
*以便** 快速接入 IAM 鉴权体系。

## 验收标准

1. `IamSdkAutoConfig` 使用 `@AutoConfiguration` + `@ComponentScan` 自动注册 SDK 包下所有组件
2. 通过 `@ConditionalOnProperty(prefix = "sh.iam.sdk", name = "enabled", havingValue = "true", matchIfMissing = true)`
   控制启用
3. 注册在 `META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports`
4. `IamSdkConfig` 集中管理所有配置项：
    - `iam.sdk.enabled`：SDK 启用开关（默认 true）
    - `iam.sdk.app-code`：应用编码
    - `iam.sdk.jwt.secret-key`：JWT 签名密钥
    - `iam.sdk.server-url`：SSO 服务端地址
    - `iam.sdk.app-id`：应用 ID（AK 鉴权）
    - `iam.sdk.app-secret`：应用密钥（AK 鉴权）
    - `iam.sdk.static.enabled`：静态资源过滤开关
    - `iam.sdk.static.subfix`：静态资源后缀
5. `ConfigValidator` 在生产环境（prod/uat/pre）强制校验 JWT 密钥、AK 凭证、服务端地址
6. 校验规则：JWT 密钥非空且长度 >= 32、AK 凭证非空且非 "default"、服务端地址以 http:// 或 https:// 开头

## 技术实现要点

- Spring Boot 3.x 自动配置注册方式（非 spring.factories）
- `@ComponentScan(basePackages = {"com.wkclz.iam.sdk"})` 扫描整个 SDK 包
- 配置键前缀注意：自动配置用 `sh.iam.sdk`，配置属性用 `iam.sdk`
- `@PostConstruct validateConfig()` 在 Bean 初始化后校验配置
- ConfigValidator 仅在 prod/uat/pre 环境强制校验，dev/test 环境不校验
