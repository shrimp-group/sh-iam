## 用户故事

**作为** 系统管理员，**我希望** 系统启动时自动扫描所有带 `@Router` 注解的 REST 端点并同步到数据库，**以便** API
权限配置与代码保持一致，无需手动维护。

## 验收标准

1. `RestfulScan` 实现 `ApplicationRunner`，应用启动时自动执行
2. 开关控制：读取 `IamAdminConfig.apiScanEnabled`，非 1 则跳过
3. 数据采集：从数据库获取全部 IamApi 记录，通过 `RestHelper.getMapping()` 扫描系统端点
4. 差异对比（以 `method_uri` 为 key）：
    - 新增：系统中有但数据库没有的 API → 生成 apiCode，根据配置决定是否自动插入
    - 更新：两边都有但 apiName 或 module 不一致的 → 自动更新
    - 删除：数据库有但系统中没有的 → 仅输出日志，不自动删除
5. 日志输出：插入/更新/删除的数量及详细清单
6. 删除时输出可直接执行的 SQL
7. `POST /iam-admin/api/sync` — 手动触发 API 自动扫描同步

## 技术实现要点

- `@Component` + `ApplicationRunner`，启动时自动执行
- apiCode 生成：`RedisIdGenerator.generateIdWithPrefix("api_")`
- 配置项 `IamAdminConfig.apiScanInsert`：1=自动插入新 API，其他=仅输出日志
- 删除操作不自动执行，需手动执行 SQL，避免误删
- 基于 `@Router` 注解扫描，Route 接口中的路由常量都会被识别
