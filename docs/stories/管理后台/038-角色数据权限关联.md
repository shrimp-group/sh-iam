## 用户故事

**作为** 系统管理员，**我希望** 管理角色与数据权限维度的关联关系，**以便** 为角色分配行级数据权限。

## 验收标准

1. `GET /iam-admin/role-data/list` — 按 roleCode + dimensionCode 查询角色数据权限列表（两者必填）
2. `POST /iam-admin/role-data/bind` — 绑定角色与数据权限（appCode、roleCode、dimensionCode、dataCode 必填）
3. `POST /iam-admin/role-data/unbind` — 解绑角色与数据权限（按 ID 删除）
4. 绑定时校验 appCode、roleCode、dimensionCode、dataCode 非空

## 技术实现要点

- 角色-数据权限通过 iam_role_data 中间表关联
- roleCode + dimensionCode 确定角色在某个维度下的数据权限
- dataCode 标识具体的数据范围值
- 与 IamDataDimension 配合实现行级数据权限控制
