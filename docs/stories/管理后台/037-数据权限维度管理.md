## 用户故事

**作为** 系统管理员，**我希望** 在管理后台定义和管理数据权限维度，**以便** 实现行级数据权限控制。

## 验收标准

1. `GET /iam-admin/data-dim/page` — 数据维度分页查询
2. `GET /iam-admin/data-dim/info` — 按 ID 查数据维度详情
3. `POST /iam-admin/data-dim/create` — 创建数据维度
4. `POST /iam-admin/data-dim/update` — 更新数据维度
5. `POST /iam-admin/data-dim/remove` — 删除数据维度
6. `GET /iam-admin/data-dim/options` — 获取全部数据维度列表（下拉选择用）
7. 参数校验：dimensionName、appCode 必填；更新时 version 必填
8. 支持两种数据来源：dimensionDataJson（静态 JSON）和 dimensionScript（动态脚本）

## 技术实现要点

- dimensionDataJson 优先级高于 dimensionScript
- dimensionDataJson 适用于固定的数据范围（如部门列表 [1,2,3]）
- dimensionScript 适用于动态数据范围（如 SQL 查询脚本），运行时执行获取数据
- 通过 appCode 按应用隔离，不同应用可定义不同的数据权限维度
- options 接口返回全部数据维度列表供下拉选择
