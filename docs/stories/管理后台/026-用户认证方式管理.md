## 用户故事

**作为** 系统管理员，**我希望** 管理用户的认证方式（密码、LDAP 等），**以便** 控制用户可用的登录方式。

## 验收标准

1. `GET /iam-admin/user-auth/list` — 按条件查询用户认证方式列表
2. `GET /iam-admin/user-auth/info` — 按 ID 查询认证方式详情
3. `POST /iam-admin/user-auth/create` — 创建用户认证方式
4. `POST /iam-admin/user-auth/update` — 更新用户认证方式
5. `POST /iam-admin/user-auth/remove` — 删除用户认证方式
6. 支持多种认证类型：PASSWORD、LDAP
7. authStatus 控制认证方式启用/禁用

## 技术实现要点

- 使用 `IamUserAuthDto` 而非 Entity 作为接口参数
- 校验逻辑在 Service 层
- 一个用户可以有多种认证方式
- authStatus：0-禁用，1-启用
