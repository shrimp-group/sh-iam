## 业务规则

### 用户管理规则

1. **用户名唯一性**：用户名在系统中必须唯一
2. **用户状态**：
   - 启用：正常登录使用
   - 禁用：无法登录，保留数据
   - 锁定：因多次登录失败被锁定
3. **密码策略**：
   - 密码过期时间：默认180天
   - 密码历史：保留最近N次密码记录
   - 禁止重复使用历史密码

### 角色管理规则

1. **角色继承**：支持父子角色继承
2. **应用隔离**：角色属于特定应用
3. **权限继承**：子角色继承父角色权限

### 菜单管理规则

1. **层级结构**：菜单支持多级树形结构
2. **类型区分**：菜单分为菜单(MENU)和按钮(BUTTON)
3. **应用隔离**：菜单属于特定应用

### 权限校验规则

1. **登录校验**：登录接口需要验证码校验
2. **Token校验**：非公开接口需要Token
3. **权限校验**：根据用户角色判断访问权限
4. **数据权限**：根据角色数据维度过滤数据

### 安全规则

1. **密码加密**：前端RSA加密，后端MD5+salt存储
2. **Token安全**：JWT + Redis Session双重验证
3. **防暴力破解**：登录失败次数限制和锁定机制
4. **请求日志**：记录所有API请求日志

## 数据流模型

### 用户登录数据

| 数据项 | 类型 | 来源 | 用途 |
|--------|------|------|------|
| username | String | 用户输入 | 登录凭证 |
| password | String | 用户输入(RSA加密) | 登录凭证 |
| captcha | String | 用户输入 | 验证码 |
| captchaKey | String | 服务端生成 | 验证码标识 |
| ip | String | 请求头 | 登录日志 |

### 用户会话数据

| 数据项 | 类型 | 存储位置 | 有效期 |
|--------|------|---------|--------|
| token | String | JWT | 配置决定 |
| userSession | Object | Redis | 配置决定 |
| userJwt | Object | ThreadLocal | 请求级别 |

### 权限数据

| 数据项 | 类型 | 说明 |
|--------|------|------|
| menus | List | 用户可见菜单列表 |
| buttons | List | 用户可用按钮列表 |
| apis | List | 用户可访问API列表 |
| dataDimensions | List | 用户数据权限维度 |
