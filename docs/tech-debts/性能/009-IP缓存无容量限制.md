# TD-009: IP_ADDR_CACHE 无容量限制和过期策略

## 元信息

| 字段   | 值                  |
|------|--------------------|
| ID   | TD-009             |
| 分类   | PERF / memory-leak |
| 严重程度 | high               |
| 状态   | open               |
| 发现日期 | 2026-06-19         |

## 描述

IpLocalCacheHelper 中的 IP_ADDR_CACHE 使用 ConcurrentHashMap，无容量限制和过期策略。长期运行将导致内存泄漏，每个唯一 IP
地址都会永久缓存。

## 影响范围

- iam-common (IpLocalCacheHelper)
- 长期运行的 SSO 服务

## 复现条件

1. 系统长期运行
2. 大量不同 IP 地址的请求
3. IP_ADDR_CACHE 持续增长，最终导致 OOM

## 当前解决方案

无容量限制。

## 建议解决方案

使用 Guava LoadingCache 替代 ConcurrentHashMap，设置 maximumSize(10000) 和 expireAfterWrite(24h)。

## 关联模块

iam-common (IpLocalCacheHelper)
