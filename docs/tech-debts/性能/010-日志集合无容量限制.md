# TD-010: LOGS_SET 无容量限制

## 元信息

| 字段   | 值                  |
|------|--------------------|
| ID   | TD-010             |
| 分类   | PERF / memory-leak |
| 严重程度 | high               |
| 状态   | open               |
| 发现日期 | 2026-06-19         |

## 描述

LoggingFilter 中的 LOGS_SET 使用 HashMap，无容量限制，每个唯一 URI 都会缓存判断结果。长期运行内存持续增长。

## 影响范围

- iam-sdk (LoggingFilter)
- 所有使用 iam-sdk 的应用

## 复现条件

1. 应用长期运行
2. 大量不同 URI 的请求
3. LOGS_SET 持续增长，最终导致 OOM

## 当前解决方案

无容量限制。

## 建议解决方案

使用 Guava Cache 或 Caffeine 替代 HashMap，设置 maximumSize(1000) 和 expireAfterWrite(1h)。

## 关联模块

iam-sdk (LoggingFilter)
