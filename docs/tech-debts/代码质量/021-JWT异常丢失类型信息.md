# TD-021: JWT 异常包装丢失类型信息

## 描述

JwtUtil.parseClaims() 将 5 种 JWT
异常（ExpiredJwtException、UnsupportedJwtException、MalformedJwtException、SignatureException、IllegalArgumentException）统一包装为
new Exception("中文描述", e)，丢失了异常类型信息，调用方无法区分过期、签名错误等不同情况。

## 影响范围

- iam-sdk (JwtUtil)
- 所有调用 JwtUtil 的鉴权逻辑

## 复现条件

1. JWT 过期或签名错误
2. 调用方捕获 Exception 无法区分具体原因
3. 无法针对不同错误返回不同响应

## 当前解决方案

统一包装为 Exception。

## 建议解决方案

1. 定义自定义异常类（如 JwtValidationException），保留原始异常类型
2. 或直接抛出原始异常，由调用方处理
3. 至少区分 Token 过期和 Token 无效两种情况
