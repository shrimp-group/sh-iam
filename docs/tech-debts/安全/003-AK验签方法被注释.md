# TD-003: AK 签名验签方法被注释

## 元信息

| 字段   | 值                  |
|------|--------------------|
| ID   | TD-003             |
| 分类   | SEC / auth-missing |
| 严重程度 | high               |
| 状态   | open               |
| 发现日期 | 2026-06-19         |

## 描述

AkSignHelper 中的 deSign() 验签方法被完整注释掉，RedisLock 依赖也被注释。SsoFacadeImpl 使用 AK 签名发送请求，但服务端无法验证签名，AK
签名机制形同虚设。

## 影响范围

- iam-sdk (AkSignHelper)
- 所有使用 AK 鉴权的第三方应用
- API 调用安全性

## 复现条件

1. 第三方应用使用 AK 发起请求
2. 请求中携带 AK 签名
3. 服务端无法验证签名，签名校验被跳过
4. 攻击者可伪造请求绕过 AK 鉴权

## 当前解决方案

无。签名验证功能被注释。

## 建议解决方案

1. 实现 deSign() 方法，启用签名验证
2. 启用 RedisLock 防重放攻击
3. 在 IamAuthFilter 中增加 AK 签名校验逻辑

## 关联模块

iam-sdk (AkSignHelper)
