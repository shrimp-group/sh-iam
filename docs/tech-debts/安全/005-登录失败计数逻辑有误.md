# TD-005: 登录失败次数检查逻辑有误

## 元信息

| 字段   | 值           |
|------|-------------|
| ID   | TD-005      |
| 分类   | SEC / owasp |
| 严重程度 | medium      |
| 状态   | open        |
| 发现日期 | 2026-06-19  |

## 描述

SsoLoginLogMapper 的 getLoginFailedCountIn1Hour 方法名暗示获取失败次数，但实际只返回最近一条登录记录的 loginStatus，且判断条件
loginStatus != 0 不精确。

## 影响范围

- iam-sso (IamLoginService, SsoLoginLogMapper)

## 复现条件

1. 用户连续登录失败多次
2. 系统调用 getLoginFailedCountIn1Hour 判断是否需要验证码
3. 由于只检查最近一条记录，可能无法正确触发验证码保护

## 当前解决方案

方法存在但逻辑不正确。

## 建议解决方案

1. 修正方法实现，真正统计 1 小时内登录失败次数
2. 使用 COUNT + WHERE 条件统计失败记录数

## 关联模块

iam-sso (IamLoginService, SsoLoginLogMapper)
