# TD-002: 密码使用 MD5+salt 加密不符合现代安全标准

## 元信息

| 字段   | 值           |
|------|-------------|
| ID   | TD-002      |
| 分类   | SEC / owasp |
| 严重程度 | critical    |
| 状态   | open        |
| 发现日期 | 2026-06-19  |

## 描述

密码使用 MD5+salt 方式加密存储。MD5 已被证明存在碰撞攻击，且计算速度过快（约 10 亿次/秒），容易被暴力破解。现代安全标准要求使用
BCrypt、SCrypt 或 Argon2 等慢速哈希算法。STORY-003 要求实现 BCrypt + MD5 双算法兼容及 upgradePassword() 迁移路径，但实际仅实现了
MD5+salt。

## 影响范围

- iam-common (PasswordHelper)
- iam-sso (IamLoginService)
- iam-admin (IamUserService)
- 所有用户密码数据

## 复现条件

1. 使用 GPU 加速工具对 MD5 哈希进行暴力破解
2. 每秒可尝试数十亿次，即使有 salt 也能在合理时间内破解弱密码

## 当前解决方案

使用 salt 增加破解难度，但 MD5 的计算速度仍然过快。

## 建议解决方案

1. 实现 STORY-003 要求的 BCrypt 双算法兼容
2. PasswordHelper 增加 verifyPassword() 方法，先尝试 BCrypt 验证，失败后尝试 MD5+salt
3. 增加 upgradePassword() 迁移路径，用户登录成功后自动将 MD5 密码升级为 BCrypt
4. 新创建用户强制使用 BCrypt

## 关联模块

iam-common (PasswordHelper), iam-sso (IamLoginService), iam-admin (IamUserService)
