# TD-001: JWT 密钥硬编码默认值

## 元信息

| 字段   | 值                  |
|------|--------------------|
| ID   | TD-001             |
| 分类   | SEC / auth-missing |
| 严重程度 | critical           |
| 状态   | open               |
| 发现日期 | 2026-06-19         |

## 描述

IamSdkConfig 中 jwtSecretKey 默认值为 `qwertyuioplkjhgfdsazxcvbnmqwertyuioplkjhgfdsazxcvbnm`，如果部署时未覆盖配置，所有
JWT 可被伪造。任何知道此默认值的人都可以生成有效的 JWT Token，从而冒充任意用户访问系统。

## 影响范围

- iam-sdk 模块
- 所有依赖 iam-sdk 的第三方应用
- 生产环境安全性

## 复现条件

1. 部署 iam-sdk 时不配置 iam.sdk.jwt.secret-key
2. IamSdkConfig 使用默认密钥值
3. 攻击者使用默认密钥生成 JWT Token
4. 系统无法区分合法和伪造的 Token

## 当前解决方案

无。当前默认值始终存在，缺少启动时校验。

## 建议解决方案

1. 移除 jwtSecretKey 的默认值
2. 实现 STORY-013 要求的 ConfigValidator，在启动时强制校验密钥非空且长度 >= 32
3. 在 prod/uat/pre 环境下，如果密钥未配置则阻止启动

## 关联模块

iam-sdk (IamSdkConfig)
