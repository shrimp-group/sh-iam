# TD-016: Ip2LocationScheduler 使用裸线程

## 元信息

| 字段   | 值                  |
|------|--------------------|
| ID   | TD-016             |
| 分类   | CONC / thread-pool |
| 严重程度 | high               |
| 状态   | open               |
| 发现日期 | 2026-06-19         |

## 描述

Ip2LocationScheduler 使用 new Thread() 创建线程，无线程池管理，无优雅关闭机制。volatile running 标志在应用关闭时无人设置为
false，线程可能永远运行。

## 影响范围

- iam-sso (Ip2LocationScheduler)
- 应用关闭流程

## 复现条件

1. 应用启动后 Ip2LocationScheduler 创建线程
2. 应用关闭时线程无法正确终止
3. 可能导致应用无法正常关闭

## 当前解决方案

使用 volatile running 标志，但无人设置为 false。

## 建议解决方案

使用 Spring 的 @Scheduled 或线程池（ExecutorService）替代，确保应用关闭时线程能正确终止。实现 DisposableBean 或 @PreDestroy
进行清理。

iam-sso (Ip2LocationScheduler)
