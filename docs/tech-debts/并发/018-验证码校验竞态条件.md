# TD-018: 验证码校验与删除之间存在竞态条件

## 元信息

| 字段   | 值                     |
|------|-----------------------|
| ID   | TD-018                |
| 分类   | CONC / race-condition |
| 严重程度 | medium                |
| 状态   | open                  |
| 发现日期 | 2026-06-19            |

## 描述

IamLoginService.loginByUsernameAndPassword() 中验证码校验使用 getAndDelete 原子操作获取验证码，但登录失败次数检查与验证码删除之间无原子保证，存在竞态条件。

## 影响范围

- iam-sso (IamLoginService)

## 复现条件

1. 同一验证码被并发提交多次
2. 在失败次数检查和验证码删除之间，其他请求可能通过验证

## 当前解决方案

使用 Redis getAndDelete 保证验证码一次性使用。

## 建议解决方案

考虑使用 Redis Lua 脚本保证验证码校验和删除的原子性，将验证码校验和失败次数检查合并为一个原子操作。

## 关联模块

iam-sso (IamLoginService)
