# Grill-Me — 纯追问

和 `/grill` 一样逐条追问，但**不绑定变更目录，不写任何文件**。
适用于任何想理清的想法——技术决策、设计方案、问题分析——不需要开变更的场景。

## 核心原则

逐条追问用户以理清当前话题。每个问题附带推荐选项和理由。

## 追问规则

1. 逐条追问，每次只问一个问题
2. 每个问题附带推荐选项和理由
3. 如果问题可以通过探索代码库回答，直接探索代码库
4. 走完每个决策分支，直到达成共识

## 停止条件

- 所有决策分支理清 → 停止
- 用户中途要求停止 → 停止
